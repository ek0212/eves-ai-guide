Notes

Pricing, harness picks, benchmarks, and a couple of setup tricks.

Opus pricing

Opus 4.6 / 4.7 / 4.8 all priced at $5 input / $25 output per million tokens. Confirm current rates on each release: 4.8, 4.7, 4.6.

GPT / Codex pricing

GPT-5.5 and Codex rates change frequently across tiers and reasoning effort. Check the OpenAI pricing page or your billing dashboard for the current numbers.

CLI > app for Claude

Claude Code CLI beats the desktop app: real shell access, repo context, hooks, and the agent loop are first-class. The app is fine for chat, but coding work lives in the terminal.

Harness matters

Model is only half the choice. For getting a project off the ground end-to-end without large functionality bugs, Codex harness + GPT latest high is the most reliable path. Opus latest in Claude Code CLI (the CLI, not the desktop app) takes over once the work is deep repo reasoning.

Benchmarks worth trusting

For real datapoints on how models actually perform on code, check SWE-bench and DeepSWE bench. Vendor charts are marketing; SWE-bench-style harness evals show whether the model lands a working patch.

Research split

Use NotebookLM when you already have the source material (PDFs, papers, transcripts) and want grounded answers and audio summaries from it. Use Claude when the work is open-ended thinking, synthesis across topics, or judgment calls where no fixed source set exists.

Hotkey: cc for Claude Code

Set up a one-letter shortcut so typing cc in any Terminal tab launches Claude Code in the current directory.

  1. Open Terminal (or iTerm) on macOS.
  2. Open your shell config in a text editor: open -e ~/.zshrc (or use nano ~/.zshrc in the terminal).
  3. Add this line at the bottom of the file and save: alias cc='claude --permission-mode auto'
  4. Reload the config in your current shell: source ~/.zshrc (or open a new Terminal tab).
  5. cd into any project folder and run cc to start Claude Code.

Why --permission-mode auto: auto-approves safe, reversible operations (reads, edits in the project) but still prompts before irreversible or risky actions (deleting files, running shell commands outside the repo, network calls). Safer default than --dangerously-skip-permissions, which skips every prompt.
